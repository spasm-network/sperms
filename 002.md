# SRM-002

## Spasm Events Overview

`draft` `optional` `meta` `guardian:spasm-core`

---

### Abstract

This **SRM** explains the major differences between various types of Spasm Events.

---

### Motivation

Most protocols have one event object for convenience, but there are multiple use cases for having different object structures. For example, propagating a compressed version of an event is beneficial for the network since it decreases bandwidth, while having a fully uncompressed version with many helper keys increases developer experience.

---

### Specification

- **SpasmEventV2** - fully uncompressed version, useful for development 
- **SpasmEventBodyV2** - the main event body signed by users.
- **SpasmEventBodySignedClosedV2** - contains signed body and signature.
- **SpasmEventEnvelopeV2** - compressed version, used for sending events across the network.
- **SpasmEventEnvelopeV2** - compressed version with comments, parent and root events.
- **SpasmEventDatabaseV2** - used for saving events into a database.
- **EventForSpasmid01** - used for calculating Spasm ID.

TypeScript/JavaScript applications can use [spasm.js](https://www.npmjs.com/package/spasm.js) npm library to convert any event into SpasmEventV2.


```typescript

  DmpEvent |
  DmpEventSignedClosed |
  DmpEventSignedOpened |
  NostrEvent |
  NostrEventSignedOpened |
  NostrSpasmEvent |
  NostrSpasmEventSignedOpened |
  SpasmEventV0 |
  SpasmEventV2 |
  SpasmEventBodyV2 |
  SpasmEventEnvelopeV2 |
  SpasmEventEnvelopeWithTreeV2 |
  SpasmEventDatabaseV2 |
  SpasmEventBodySignedClosedV2

const spasmEvent: SpasmEventV2 = convertToSpasm(unknownEventV2)
```
